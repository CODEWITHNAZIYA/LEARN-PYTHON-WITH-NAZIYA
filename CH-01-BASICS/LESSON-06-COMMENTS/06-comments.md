
# Lesson 06 - Comments

## What are Comments?

Comments are notes written inside the code to explain what the code does.

Python ignores comments while executing the program.

---

# Why Do We Use Comments?

Comments make code:

- Easy to read
- Easy to understand
- Easy to maintain
- Helpful for teamwork

---

# Single-Line Comment

A single-line comment starts with `#`.

Example

```python
# This is a comment

print("Hello")
```

Output

```
Hello
```

---

# Multi-Line Comment

Multi-line comments can be written using triple quotes.

Example

```python
"""
This is a multi-line comment.
"""
```

---

# Example

```python
# Store name
name = "Naziya"

print(name)
```

---

# Good Comments

Good comments explain **why** the code is written.

Example

```python
# Calculate total marks
total_marks = 450
```

---

# Bad Comments

Avoid writing comments that repeat the code.

❌ Bad Example

```python
# Print name
print(name)
```

The code is already clear.

---

# Best Practices

✔ Write meaningful comments.

✔ Keep comments short and simple.

✔ Update comments when code changes.

✔ Use comments only when needed.

---

# Common Mistakes

❌ Writing too many unnecessary comments.

❌ Writing outdated comments.

❌ Writing comments that don't match the code.

---

# Tips

- Use comments to explain difficult logic.
- Don't comment every line.
- Keep comments simple and clear.
